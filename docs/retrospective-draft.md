# SE能力可視化アプリ、初期リリースまでの全記録 ― AIと作る開発プロセス

> ※ note メンバーシップ向け記事の下書き

---

## 1. はじめに

エンジニアの能力を、ゲームのステータス画面風に可視化するWebアプリ「**SE Ability Visualizer**」を公開しました。

🔗 https://se-ability-visualizer.vercel.app/

名前と能力値を入力すると、S〜Gのランク付きステータスカードが生成され、スクリーンショットやリンクでSNSにシェアできる、という小さなアプリです。

このアプリを作った目的は、機能そのものよりも「**開発プロセス自体をコンテンツにする**」ことにあります。AIで誰でもアプリが作れる時代だからこそ、完成品ではなく「どう作ったか・どこでつまずいたか」という過程に価値があると考えました。

この記事では、企画からリリースまでの流れを、**うまくいかなかったことも含めて**正直に振り返ります。これから個人開発をしたい人、AIを使った開発に興味がある人の参考になればと思います。

---

## 2. 何を作ったのか

機能はあえてシンプルに絞りました。

- 基本情報の入力（表示名・タイプ名・コメント）
- メイン能力 7項目をスライダーで入力（開発力・設計力・インフラ力・DB力・リード力・マネジメント力・AI活用力）
- 特殊能力 20項目をタグで選択
- 入力値に応じた S〜G ランク表示
- ステータスカード風の結果画面

逆に、**あえて作らなかったもの**を最初に決めました。

- ログイン / データベース保存
- AI自動診断（能力を自動でスコアリングする機能）
- PDF出力 / 管理画面

> **学び①：最初に「作らないもの」を決める**
>
> やりたいことは無限に出てきます。MVP（最小限の製品）では「何を作るか」より「何を作らないか」を先に線引きするほうが、迷わず最後まで走り切れました。

## 3. 技術構成と、その選定理由

| 項目 | 技術 |
|------|------|
| フレームワーク | React + Vite + TypeScript |
| スタイル | Tailwind CSS |
| スマホ対応 | PWA（ホーム画面に追加可能） |
| デプロイ | Vercel |

選定の軸は「**運用が軽いこと**」でした。

- バックエンドを持たない（入力内容はブラウザのlocalStorageに保存）
- 無料で公開・維持できる
- Webアプリとして作りつつ、PWAでスマホアプリのようにも使える

> **学び②：個人開発は「運用が軽い構成」を選ぶ**
>
> 個人開発は、作って終わりではなく「維持」が続きます。サーバー代やDB管理が発生しない構成にしておくと、公開後の負担がぐっと減ります。

## 4. 開発の進め方 ― AIとの協働

開発は、AIエージェント（Claude Code）と一緒に進めました。流れはこうです。

1. Notionでタスクを細かく分解（型定義 → コンポーネント → 画面 → デプロイ…）
2. 1タスクずつAIに依頼して実装
3. ビルド・動作確認・GitHubのPR作成までAIが実行
4. 自分はレビューと方向性の判断に集中

印象的だったのは、**「実装そのもの」より「何をどう頼むか」と「ちゃんと動いたかの確認」**に時間の価値が移ったことです。

> **学び③：AI時代は「指示の設計」と「確認の仕組み」が価値になる**
>
> コードを書く速度はAIで上がります。だからこそ、タスクをどう分解し、どう検証するか ― 人間側の段取りが品質を決めます。

## 5. つまずきポイント（本音パート）

ここが一番リアルな部分です。「動いて当然」と思っていたところほど、つまずきました。

### ① デプロイが自動で反映されない

GitHubにマージすればVercelが自動デプロイしてくれる ― はずでした。ところがマージしても本番が一向に更新されない。原因はGitHub連携が正しくデプロイをトリガーできていなかったこと。

結局、Vercel CLIから手動でデプロイ（`vercel --prod`）して回避しました。「自動化されているはず」が崩れたときに、手動の代替手段を知っておくことの大切さを痛感しました。

### ② AIが生成した画像の「白い背景」問題

アプリのアイコンやボタンはAIで画像生成しました。ところが画面に置くと、ボタンの周りに**白い四角い箱**が表示される。透過PNGに見えて、実は背景が白く塗りつぶされていたのが原因でした。

画像の縁から白い部分だけを透明に変換する処理（フラッドフィル）をかけて解決。AI生成素材は「そのまま使える」とは限らない、という学びでした。

### ③ PWAアイコンの参照切れ

「スマホのホーム画面に追加」したときのアイコンが表示されない。調べると、設定ファイル（manifest）は `icon-192.png` を参照しているのに、**実ファイルが存在しなかった**。テンプレートの初期設定が残ったままだったのです。

> **学び④：動いて当然のところほど落とし穴がある**
>
> 派手な機能より、デプロイ・画像・設定ファイルといった「土台」でつまずく。逆に言えば、ここを乗り越える過程こそが、再現性のある一次情報になります。

## 6. リリースしてみて

公開して終わりではなく、「**反応を見られる状態**」を先に作りました。

- **アクセス解析**（Vercel Analytics）：何人が来て、どこまで使ったかを計測
- **OGP対応**：XやnoteにURLを貼ったときに、ロゴ入りのリンクカードが出るように
- **共有導線**：結果画面に「シェアする」ボタンを設置（スマホのネイティブ共有・Xシェア・リンクコピー）

機能を増やす前に「使われたかが分かる仕組み」を整えたことで、次に何を改善すべきかを**データで判断できる土台**ができました。
